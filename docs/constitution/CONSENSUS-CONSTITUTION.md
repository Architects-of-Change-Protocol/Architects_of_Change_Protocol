# Consensus Constitution

**Constitution Version:** v1.0

## Constitutional purpose

**Consensus** is a governed determination that sufficient constitutional agreement exists regarding a claim, verification, reputation, attestation, proposal, decision, or constitutional artifact. Consensus answers **whether enough constitutionally valid actors stand behind this**. Consensus is not authority, capability, policy, standing, claim, trust, verification, reputation, attestation, or decision.

## Sovereign consensus rule

A proposal may have attestations, trust signals, and verification, but only a consensus type cataloged in `CONSENSUS-AUTHORITIES.md`, computed under its consensus model, satisfying its threshold policy, maintained through the constitutional lifecycle, and expiring per its expiration policy may constitute governed agreement.

The constitutional chain is:

`Authority → Capability → Policy → Standing → Claim → Trust → Verification → Reputation → Attestation → Consensus → Decision → Action`

Consensus may influence a decision only as a traceable collective agreement input. It never replaces required standing, evidence, policy coverage, trust evaluation, verification requirements, reputation signals, attestation endorsements, decision authority, or explainability, and it never directly authorizes action.

## Consensus legitimacy

A consensus is legitimate only when all of the following are traceable:

1. a canonical consensus authority and owner;
2. a consensus model satisfying the model policy for that consensus type;
3. sufficient attestations satisfying the threshold policy;
4. a valid lifecycle path through `Collecting` before `Established`;
5. every expiration event, dispute, revocation, and retirement; and
6. the current consensus status separately from preserved historical records.

No consensus may become `Established` without threshold satisfaction. No consensus may override standing, capability, policy, claim evidence, verification requirements, reputation signals, or attestation endorsements.

## Constitutional properties

Consensus is model-governed, threshold-bound, attestation-aggregating, purpose-bound, non-transferable, time-aware, expiring where cataloged, disputable where cataloged, revocable where cataloged, historically preserved, and constitutionally evolvable. A consensus determination never copies agreement status from one subject, context, claim, or authority to another.

## Attestation ≠ Consensus

Attestation and consensus are constitutionally independent:

| Attestations Exist | Threshold Satisfied | Consensus Established | Meaning |
|---|---|---|---|
| No | No | No | No endorsement; no agreement |
| Yes | No | No | Endorsements exist; agreement not established |
| Yes | Yes | No | Threshold met; consensus not yet evaluated |
| Yes | Yes | Yes | Governed agreement established |

All four states are constitutionally valid. Decision authorities may consume attestations and consensus as independent inputs.

## Consensus is an input, not a bypass

Consensus may influence decisions. Consensus may not override:

- Authority
- Capability
- Policy
- Standing
- Claim Evidence
- Verification Requirements
- Reputation Signals
- Attestation Endorsements

Consensus is a governed input to decision-making. It is never a constitutional bypass.

## Scope boundary

This framework governs consensus legitimacy and constitutional records only. It does not implement a voting engine, governance engine, federation engine, consensus runtime, blockchain consensus, distributed consensus protocol, or AI consensus engine.
