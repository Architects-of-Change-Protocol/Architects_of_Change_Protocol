# Verification Revocation Policy

**Constitution Version:** v18.0

## Constitutional rule

A `Verified` or `Pending Verification` verification may be revoked only by an authorized revocation authority for a valid cause. Every revocation is permanently recorded. Revoked verifications authorize no new validation influence and may not be reactivated; they must retire.

## Valid revocation causes

| Cause | Description |
|---|---|
| Fraud | Verification obtained through misrepresentation or falsified evidence |
| Evidence Failure | Supporting evidence invalidated or withdrawn |
| Method Failure | Verification method found to be unauthorized or defective |
| Claim Failure | Underlying claim rejected, withdrawn, or superseded |
| Trust Failure | Underlying trust signal revoked |
| Standing Failure | Subject's standing revoked |
| Constitutional Override | Constitutional amendment invalidates the verification |
| Governance Decision | Ratified governance decision revokes the verification |
